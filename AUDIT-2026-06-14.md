**Date:** 2026-06-14  
**Auditor:** Kimi Code CLI  
**Scope:** Human site, AI/agent APIs, payment gating, Cloudflare Worker, deployment/infrastructure  
**Method:** Codebase exploration + live HTTP testing + targeted code review

## Executive Summary

| Domain | Status | Biggest Risk |
|--------|--------|--------------|
| Human site | вљ пёЏ Mostly working | Two disabled features (interactive repair guide, diagnose chat); generic indexable pages drift |
| AI Training Feed | рџ”ґ Broken in parts | HEAD requests crash (500); Stripe checkout is not configured on the live server |
| Payment gating | рџџЎ Partial | x402 devnet is consistent, but Stripe real-money flow is dead in production |
| Agent discovery | рџџЎ Drifted | Several endpoints omit the training feed; ACP response is duplicated; JWKS stubs |
| Cloudflare Worker | рџџў Working | Bot classification and redirects function as designed |
| Infrastructure | рџ”ґ Critical | `prestart` script is malformed; secrets in tracked files; API keys stored in plaintext |

**Top 5 issues to fix immediately:**
1. Stripe environment variables are missing on the production VPS вЂ” real-money payments cannot work.
2. `package.json` `prestart` script is concatenated incorrectly and will break service restarts.
3. Paid API routes return 500 for HEAD requests (Next.js route handlers only implement GET).
4. `/.well-known/acp.json` returns duplicated/conflicting JSON.
5. `/llms.txt` is served with `Content-Type: application/xml` instead of `text/plain`.

## 2. Payments & API Gating

| # | Severity | Finding | Evidence | Recommendation |
|---|----------|---------|----------|----------------|
| 2.1 | рџ”ґ **Critical** | **Stripe checkout is non-functional in production** (same root cause as 1.1). Real-money conversions are impossible. | Live test: `503 {"error":"Stripe is not configured..."}` | Add Stripe keys; create live price IDs for the four credit packs; verify webhook endpoint is registered in Stripe dashboard. |
| 2.2 | рџ”ґ **High** | **HEAD requests to paid API routes crash with 500.** The route handlers only export `GET`; Next.js expects a `Response` for all methods. Logs show: "No response is returned from route handler ... | 2.3 | рџ”ґ **High** | **Turnstile test secret fallback disables bot protection.** If `TURNSTILE_SECRET_KEY` is unset, the test key `1x0000000000000000000000000000000AA` validates any token. | `src/app/api/generate-guide/route.ts:113` | Remove the fallback; fail closed (return 500) when the secret is missing. |
| 2.4 | рџџЎ **Medium** | **Payment gate bypass risk.** `checkStripeAccess` treats any non-Stripe-format bearer token as x402 and allows it. | 2.6 | рџџЎ **Medium** | **`/api/account?session_id=...` exposes the API key to anyone with the Checkout Session ID.** Session IDs appear in browser history and server logs. | `src/app/api/account/route.ts:61` | Return only a one-time setup token or redirect to an authenticated session; do not return the persistent API key. |
| 2.7 | рџџЎ **Medium** | **`/api/account` POST can create an orphan customer row.** `getOrCreateCustomerByEmail` inserts a new row before the caller checks whether the returned ID matches the authenticated customer. | `src/app/api/account/route.ts:175` | Move the existence check before insertion, or wrap the email update in a transaction. |
| 2.8 | рџџў **Low** | **API key format has no prefix/scoping.** 64-char hex is hard to distinguish from other hashes and cannot be rotated per-environment. | `src/lib/apiKey.ts:9` | Prefix keys (e.g., `aom_live_...`, `aom_test_...`) and store only hashes. |
